## The Multi-Threaded Tkinter Blueprint
To build this reliably, structure your Python code using a Thread -> Queue -> GUI architecture:

   1. The Main Thread (Tkinter GUI): Handles user clicks, button drawings, and updates the frequency display. It uses Tkinter’s .after() loop to safely check for incoming data. [7, 8] 
   2. The uBITX Thread: A background worker loop that constantly reads the serial port for physical VFO knob spins or S-meter changes from the CEC firmware.
   3. The SDR++ Thread: A background worker loop that listens to the SDR++ network socket (usually a TCP connection to its rigctld or control server) for waterfall clicks.

## Code Pattern: Safe Asynchronous Communication
Never update a Tkinter widget (like label.config()) directly from a background thread, or your app will randomly crash. Use Python's thread-safe queue.Queue to pass messages back to the GUI. [9, 10, 11, 12]
